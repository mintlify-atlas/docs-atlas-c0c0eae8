---
title: "Quickstart: run Hadoop locally"
description: "Download a pre-built Hadoop binary, start HDFS and YARN on a single machine, and submit your first MapReduce WordCount job in under 15 minutes."
---

This quickstart walks you through downloading a pre-built Hadoop binary, configuring it on a single machine in pseudo-distributed mode, starting the HDFS and YARN daemons, and verifying your cluster by running the bundled WordCount example. Pseudo-distributed mode runs all Hadoop daemons as separate JVM processes on one machine, giving you a fully functional cluster that behaves identically to a multi-node deployment — making it the fastest way to learn Hadoop before scaling out.

<Note>
  Hadoop requires **Java 17**. Confirm your Java version before proceeding:

  ```bash
  java -version
  ```

  You should see output containing `openjdk version "17"` or similar. If not, install OpenJDK 17 from your package manager before continuing.
</Note>

<Steps>
  <Step title="Download the Hadoop binary">
    Download the latest pre-built Hadoop release from the Apache Hadoop downloads page. Replace the version number in the commands below with the current stable release if it differs.

    ```bash
    wget https://downloads.apache.org/hadoop/common/stable/hadoop-3.4.1.tar.gz
    tar -xzf hadoop-3.4.1.tar.gz
    mv hadoop-3.4.1 /opt/hadoop
    ```

    <Tip>
      Verify the download against the published SHA-512 checksum before extracting in production environments. The checksum file is available alongside the tarball on the Apache downloads page.
    </Tip>

    After extraction you will have a directory layout like this:

    ```
    /opt/hadoop/
    ├── bin/          # hadoop, hdfs, yarn, mapred CLIs
    ├── sbin/         # daemon start/stop scripts
    ├── etc/hadoop/   # configuration files
    ├── share/hadoop/ # JARs, including example jobs
    └── lib/          # native libraries
    ```
  </Step>

  <Step title="Set JAVA_HOME">
    Hadoop needs to know where Java is installed. Set `JAVA_HOME` in `etc/hadoop/hadoop-env.sh` — this is the authoritative location and overrides any shell environment variable.

    Open `/opt/hadoop/etc/hadoop/hadoop-env.sh` and find the `JAVA_HOME` line. Uncomment it and set it to your JDK path:

    ```bash
    export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
    ```

    <Tabs>
      <Tab title="Ubuntu / Debian">
        ```bash
        # Find your JDK path
        update-java-alternatives --list

        # Then set it in hadoop-env.sh:
        export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
        ```
      </Tab>
      <Tab title="macOS (Homebrew)">
        ```bash
        # Find your JDK path
        /usr/libexec/java_home -v 17

        # Then set it in hadoop-env.sh:
        export JAVA_HOME=/opt/homebrew/opt/openjdk@17
        ```
      </Tab>
      <Tab title="Rocky Linux / RHEL">
        ```bash
        # Find your JDK path
        alternatives --list | grep java

        # Then set it in hadoop-env.sh:
        export JAVA_HOME=/usr/lib/jvm/java-17-openjdk
        ```
      </Tab>
    </Tabs>

    Also add Hadoop's `bin` directory to your shell's `PATH` for convenience:

    ```bash
    export HADOOP_HOME=/opt/hadoop
    export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
    ```

    Add these lines to your `~/.bashrc` or `~/.zshrc` and run `source ~/.bashrc` to apply them.
  </Step>

  <Step title="Configure pseudo-distributed mode">
    Three configuration files need minimal edits to enable pseudo-distributed mode. All files are in `/opt/hadoop/etc/hadoop/`.

    **`core-site.xml`** — tells Hadoop to use a local HDFS instance as the default file system:

    ```xml core-site.xml
    <configuration>
      <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
      </property>
    </configuration>
    ```

    **`hdfs-site.xml`** — sets replication to 1 (appropriate for a single-node setup):

    ```xml hdfs-site.xml
    <configuration>
      <property>
        <name>dfs.replication</name>
        <value>1</value>
      </property>
    </configuration>
    ```

    **`mapred-site.xml`** — tells MapReduce to use YARN as the execution framework:

    ```xml mapred-site.xml
    <configuration>
      <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
      </property>
      <property>
        <name>mapreduce.application.classpath</name>
        <value>$HADOOP_MAPRED_HOME/share/hadoop/mapreduce/*:$HADOOP_MAPRED_HOME/share/hadoop/mapreduce/lib/*</value>
      </property>
    </configuration>
    ```

    **`yarn-site.xml`** — enables the MapReduce shuffle service on NodeManagers:

    ```xml yarn-site.xml
    <configuration>
      <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
      </property>
    </configuration>
    ```
  </Step>

  <Step title="Configure SSH for localhost">
    The Hadoop start scripts use SSH to launch daemons. On a single node you need passwordless SSH to `localhost`:

    ```bash
    # Install openssh-server if not already present
    sudo apt-get install -y openssh-server   # Ubuntu/Debian
    # sudo dnf install -y openssh-server     # Rocky Linux

    # Generate an SSH key if you don't have one
    ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

    # Authorize it for localhost login
    cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
    chmod 0600 ~/.ssh/authorized_keys

    # Verify passwordless login works
    ssh localhost
    ```

    If `ssh localhost` connects without prompting for a password, you are ready to proceed.
  </Step>

  <Step title="Format the NameNode">
    Before starting HDFS for the first time you must format the NameNode. This initializes the on-disk metadata structures (the FSImage) in the directory configured by `dfs.namenode.name.dir` (defaults to `/tmp/hadoop-$USER/dfs/name`).

    ```bash
    hdfs namenode -format
    ```

    Successful output includes a line like:

    ```
    Storage directory /tmp/hadoop-<user>/dfs/name has been successfully formatted.
    ```

    <Warning>
      Running `hdfs namenode -format` on an existing cluster **destroys all namespace metadata**. Only run this command when setting up a fresh cluster.
    </Warning>
  </Step>

  <Step title="Start HDFS">
    Start the NameNode and DataNode daemons using the provided script:

    ```bash
    start-dfs.sh
    ```

    You should see output indicating that the NameNode, DataNode, and Secondary NameNode have been started. Verify with the `jps` (Java process status) command:

    ```bash
    jps
    ```

    Expected output:

    ```
    12345 NameNode
    12456 DataNode
    12567 SecondaryNameNode
    12678 Jps
    ```

    You can also inspect the HDFS web UI at `http://localhost:9870` to confirm the cluster is healthy.

    ```bash
    # Create a home directory in HDFS for your user
    hdfs dfs -mkdir -p /user/$USER
    ```
  </Step>

  <Step title="Start YARN">
    Start the ResourceManager and NodeManager daemons:

    ```bash
    start-yarn.sh
    ```

    Verify with `jps`:

    ```bash
    jps
    ```

    Expected output now includes:

    ```
    12345 NameNode
    12456 DataNode
    12567 SecondaryNameNode
    12789 ResourceManager
    12890 NodeManager
    12901 Jps
    ```

    The YARN ResourceManager web UI is available at `http://localhost:8088`.
  </Step>

  <Step title="Verify HDFS with basic commands">
    Confirm HDFS is working by creating a directory and listing the root:

    ```bash
    # List the root of HDFS
    hadoop fs -ls /

    # Create a directory for input data
    hadoop fs -mkdir -p /user/$USER/input

    # Upload a local file to HDFS
    echo "the quick brown fox jumps over the lazy dog" > /tmp/sample.txt
    hadoop fs -put /tmp/sample.txt /user/$USER/input/

    # Confirm the file was stored
    hadoop fs -ls /user/$USER/input/
    ```

    You should see `sample.txt` listed with its size, replication factor, and timestamps.
  </Step>

  <Step title="Submit the WordCount example job">
    Hadoop ships with a collection of example MapReduce jobs in `share/hadoop/mapreduce/`. Run the WordCount example against the file you just uploaded:

    ```bash
    hadoop jar \
      share/hadoop/mapreduce/hadoop-mapreduce-examples-*.jar \
      wordcount \
      /user/$USER/input \
      /user/$USER/output
    ```

    You will see log output from the MapReduce ApplicationMaster showing map and reduce progress:

    ```
    ...
    INFO mapreduce.Job: map 0% reduce 0%
    INFO mapreduce.Job: map 100% reduce 0%
    INFO mapreduce.Job: map 100% reduce 100%
    INFO mapreduce.Job: Job job_XXXX_0001 completed successfully
    ...
    ```

    <Note>
      If the job already ran and `/user/$USER/output` exists, delete it first: `hadoop fs -rm -r /user/$USER/output`. MapReduce refuses to write to an existing output directory.
    </Note>
  </Step>

  <Step title="Verify the output">
    Read the output file to confirm the job ran correctly:

    ```bash
    hadoop fs -cat /user/$USER/output/part-r-00000
    ```

    Expected output (word counts for the sample sentence):

    ```
    brown	1
    dog	1
    fox	1
    jumps	1
    lazy	1
    over	1
    quick	1
    the	2
    ```

    You can also list all output files:

    ```bash
    hadoop fs -ls /user/$USER/output/
    ```

    The `_SUCCESS` file indicates the job completed without errors.
  </Step>
</Steps>

## Stopping the cluster

When you are done, stop the daemons cleanly to release resources:

```bash
stop-yarn.sh
stop-dfs.sh
```
